# Dodomoney

Dodomoney（钱多多）是一款 AI 智能记账软件。第一版目标不是做一个只有页面的演示，而是搭建可扩展的软件骨架：桌面客户端负责交互，后端服务负责账单、预算、AI 解析、数据分析与持久化。

## 项目定位

- 自然语言记账：用户输入“午饭 22 元”，系统自动识别金额、类别、时间与备注。
- AI 财务助手：默认助手“账小喵”，支持不同语气和角色风格。
- 账单管理：支持收支记录的增删改查、分类纠错和后续学习。
- 预算分析：支持月度统计、分类占比、预算提醒与异常消费分析。
- 图片扩展：后续接入小票 OCR、支付截图识别、食物识别与热量估算。

## 技术架构

```text
Dodomoney
├─ apps
│  ├─ api       FastAPI 后端服务，负责业务逻辑、数据库、AI 编排
│  └─ desktop   Electron + React 桌面客户端
├─ docs         架构、路线图、小组协作说明
├─ shared       前后端共享契约、类型与分类字典
└─ scripts      开发辅助脚本
```

## 快速开始

### 1. 启动后端

```powershell
cd apps/api
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m app.main
```

后端默认运行在 `http://127.0.0.1:8000`。

### 2. 启动桌面端

```powershell
cd apps/desktop
npm install
npm run dev
```

桌面端开发模式会启动 Vite 和 Electron。

## 小组分工建议

- 后端组：FastAPI 接口、SQLite 数据模型、预算和统计服务。
- AI 组：自然语言解析、分类规则、提示词、后续模型接入。
- 桌面端组：聊天式记账界面、账单列表、统计页、设置页。
- 产品与测试组：需求拆解、交互文案、测试用例、演示数据。

## 当前版本状态

当前版本已经形成可运行的第一版软件闭环：

- 可运行的后端入口与接口分层。
- 自然语言记账：自动提取金额、收支类型、分类、描述并落库。
- 账单明细：查询、删除，预留编辑接口；分类偏好会记录到数据库。
- 月度统计：收入、支出、结余、分类占比。
- 月度预算：创建、删除，并预留按分类预算扩展。
- AI 助手设置：五种回复风格，本地持久化选择。
- Electron 桌面端：聊天、明细、统计、设置四个工作区。
- 可替换的 AI 解析边界：当前使用规则解析，后续可接 OpenAI、通义或本地模型。

## 团队开发约定

- 后端接口变更同步更新 `apps/api/app/domain/schemas.py` 与 `shared/contracts`。
- 新功能优先落在 service 层，路由只负责参数校验和 HTTP 状态码。
- 桌面端不直接操作数据库，只通过 `apiClient` 调用后端。
- 每个成员从 `main` 创建自己的功能分支，提交信息使用 `feat:`、`fix:`、`test:` 前缀。
- 合并前至少执行 `python -m compileall -q app`，前端依赖安装后执行 `npm run build`。

后续优先级建议：接入真实模型解析并保留规则解析兜底；完善账单编辑和分页筛选；增加 OCR、图片和语音入口；再做用户登录、数据导出与多设备同步。
